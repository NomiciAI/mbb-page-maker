AI application software investment thesis

Where to invest as enterprise AI value migrates from model access to workflow ownership, data advantage, and measurable adoption.

Investment answer

The attractive AI application investments will own narrow workflows with proprietary feedback loops

1Models are not the moat

Foundation model capability is diffusing quickly; enduring value shifts toward workflow data, distribution, and evaluation.

2Workflow depth wins

Applications that change the operating process capture more budget than horizontal assistants that sit beside work.

3ROI must be observable

Buyers are moving from experimentation budgets to measured productivity, quality, revenue, and risk outcomes.

4Vertical wedge matters

Regulated, document-heavy, high-labor workflows create the strongest willingness to pay and defensibility.

5GTM quality separates winners

Enterprise AI vendors need services-like onboarding and product-led proof without losing software margins.

6Invest selectively

Prioritize category leaders in compliance operations, revenue operations, care operations, and engineering productivity.

Thesis: application-layer value accrues to workflow control plus proprietary feedback data

Diligence roadmap

A four-step diligence path should test moat quality before scaling financial underwriting

Week 1Map category boundaries

Define workflow scope, buyer budget, platform adjacency, and substitution risk.

Week 2Test customer proof

Validate ROI claims, adoption depth, renewal drivers, and implementation burden.

Week 3Assess product moat

Review data flywheel, evaluation assets, integrations, model strategy, and security posture.

Week 4Underwrite returns

Build cases for growth, margin path, capital needs, exit comparables, and downside risks.
