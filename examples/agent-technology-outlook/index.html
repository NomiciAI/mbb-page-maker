Architecture shift

Agentic systems should be designed as a spectrum from workflows to delegated agents

1Augmented LLM

Model plus retrieval, tools, memory, and instructions for one task context.

2Workflow

LLM and tools orchestrated through a defined path with clear control logic.

3Agent

System dynamically chooses steps and tools within bounded goals and policies.

4Multi-agent system

Specialized agents coordinate across tasks, systems, and organizational boundaries.

5Work system

Agents become part of the operating model, metrics, controls, and learning loop.

Design principle

Autonomy should be earned by evidence. Start with deterministic workflows where control is valuable; add agent discretion where variability creates value.

The best architecture is not always the most autonomous architecture.

Interoperability

Context access and agent-to-agent coordination become architecture requirements

Context and tool access

Purpose

Give agents governed access to enterprise systems

Reusable connectors reduce one-off integrations and make tool permissions easier to inspect.

Design job

Separate permissions from prompts

Agents should receive only the tools, data, and user context needed for the workflow.

Agent collaboration

Purpose

Let specialized agents discover and delegate work

Interoperability matters when customer, finance, engineering, and support agents need to coordinate.

Design job

Standardize contracts before scaling agents

Capabilities, task state, handoff semantics, identity, and audit trails need a common layer.

Maturity curve

The adoption path should progress from copilots to supervised workflows before operating-layer scale

Wave 1Assist work

Copilots help individuals draft, search, summarize, and reason inside familiar tools.

Wave 2Run workflows

Agents execute bounded tasks with defined tools, state, review gates, and rollback paths.

Wave 3Coordinate work

Specialized agents hand off tasks, share context, and orchestrate multi-step processes.

Wave 4Operate systems

Agents become governed components of the operating model, monitored like digital labor.

Operating model

Agent releases need a measured loop that combines engineering, risk, and workflow ownership

1Classify work

Define task criticality, data sensitivity, user group, and autonomy ceiling.

2Build workflow

Connect tools, context, policies, handoffs, and expected outcomes.

3Evaluate

Run golden tasks, adversarial cases, cost checks, and human review samples.

4Release

Launch behind gates with tracing, rollback paths, and adoption telemetry.

5Learn

Promote fixes, traces, and new tests into shared platform assets.
