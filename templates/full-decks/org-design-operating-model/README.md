# Org Design Operating Model

## Summary

Organization design and operating model archetype for translating strategy into roles, decision rights, governance forums, capabilities, and migration path.

## Visual traits

Mono executive operating model style, from-to shifts, capability heatmap, RACI governance grid, process rhythm, and migration roadmap.

## Use when

Use for organization design, operating model redesign, role clarity, governance, RACI, capability build, and decision-rights clarification.

## Do not use when

Do not use for pure headcount planning, workshop agendas, or technical architecture unless the main decision is how the organization should operate.

## Storyline pattern

Cover, operating model answer, design principles, current-to-target shifts, capability gaps, decision rights and governance, migration roadmap, decisions.

## Layout + component stack

Uses `from-to-multi`, `framework-map`, `heatmap`, `raci-governance-grid`, `process-flow`, `phase-roadmap`, and `decision-log`.

## Theme / assets

Theme: `assets/themes/mono.css`. No required media assets.

## Source inspiration

Source-neutral organization design and operating model patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with strategy priorities, current-state pain points, role records, decision forums, capabilities, process steps, governance cadence, and migration constraints.

## Flexible swaps

Swap heatmap for range competency chart when current/target scoring exists, replace RACI grid with framework map when roles are conceptual, and replace process flow with roadmap when timing matters.

## When to diverge

Diverge when the user needs an implementation plan, progress update, or workshop pack rather than operating model design.

## Do not copy page order/content

Do not copy the archetype page order, placeholder roles, footer text, or governance labels. Use it only to understand operating-model pacing.
