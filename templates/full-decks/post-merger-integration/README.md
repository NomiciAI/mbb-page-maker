# Post-Merger Integration

## Summary

Post-merger integration archetype for Day 1 readiness, 100-day planning, synergy tracking, governance, workstreams, risks, and integration decisions.

## Visual traits

Green integration style, value capture metrics, workstream roadmap, RACI governance grid, risk register, and decision close.

## Use when

Use for PMI, Day 1, 100-day plans, synergy tracking, integration governance, workstream planning, and merger execution updates.

## Do not use when

Do not use for pre-deal diligence, standalone acquisition decisions, or generic transformation decks unless integration execution is central.

## Storyline pattern

Cover, integration answer, value capture priorities, Day 1 readiness, workstream plan, governance and decision rights, risks, 100-day roadmap and decisions.

## Layout + component stack

Uses `metric-strip`, `phase-roadmap`, `raci-governance-grid`, `risk-matrix-register`, `comparison-table`, and `decision-log`.

## Theme / assets

Theme: `assets/themes/green.css`. No required media assets.

## Source inspiration

Source-neutral integration planning and value capture patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with deal thesis, synergy categories, Day 1 requirements, workstream records, governance roles, risks, timing, and decision asks.

## Path

`templates/full-decks/post-merger-integration/index.html`

## Related showcase patterns

`templates/showcase/operating-model-raci-governance-page.html`, `templates/showcase/roadmap-risk-decision-page.html`, `templates/showcase/steerco-progress-decision-page.html`

## Flexible swaps

Swap roadmap for milestone track when status matters, replace RACI grid with decision log when roles are incomplete, and replace synergy metrics with sensitivity grid when the value case depends on assumptions.

## When to diverge

Diverge when the audience needs diligence findings, investment decision, or transformation design rather than merger integration execution.

## Do not copy page order/content

Do not copy the archetype page order, placeholder workstreams, footer text, or integration labels. Use it only to understand PMI pacing.
