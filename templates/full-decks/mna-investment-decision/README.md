# M&A Investment Decision

## Summary

M&A and investment decision full-deck archetype for evaluating strategic rationale, target attractiveness, value creation, risks, sensitivity, and decision recommendation.

## Visual traits

Red investment committee style, rationale-first storyline, target assessment, value bridge, risk register, sensitivity grid, and decision close.

## Use when

Use for acquisition recommendations, investment committee decisions, partnership/resource allocation choices, or strategic buy/build/partner decisions.

## Do not use when

Do not use for generic vendor selection or fundraising pitch decks unless the core question is an investment decision.

## Storyline pattern

Cover, decision answer, strategic rationale, target attractiveness, value creation thesis, financial/operating evidence, risks, sensitivity, and recommendation.

## Layout + component stack

Uses `numbered-list-grid`, `comparison-table`, `waterfall`, `weighted-scorecard`, `risk-matrix-register`, `sensitivity-grid`, and `decision-log`.

## Theme / assets

Theme: `assets/themes/red.css`. No required media assets.

## Source inspiration

Source-neutral investment committee and acquisition recommendation patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with target criteria, value drivers, operating metrics, financial assumptions, synergy/risk records, scenarios, and decision asks.

## Path

`templates/full-decks/mna-investment-decision/index.html`

## Related showcase patterns

`templates/showcase/vendor-scorecard-decision-page.html`, `templates/showcase/evidence-table-implication-page.html`, `templates/showcase/roadmap-risk-decision-page.html`

## Flexible swaps

Swap value bridge for sensitivity grid when assumptions matter more than bridge math, replace scorecard with comparison table when criteria are not weighted, and add appendix pages only when the user provides backup data.

## When to diverge

Diverge when the user needs market scan, target screen, diligence workplan, or board update rather than a decision recommendation.

## Do not copy page order/content

Do not copy the archetype page order, placeholder assumptions, footer text, or target labels. Use it only to understand investment-decision pacing and component combinations.
