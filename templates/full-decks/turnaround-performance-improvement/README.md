# Turnaround Performance Improvement

## Summary

Turnaround and performance improvement archetype for diagnosing performance gaps, prioritizing immediate levers, protecting cash/control, and sequencing structural improvement.

## Visual traits

Red action style, performance gap metrics, root-cause logic, quick-win portfolio, risk controls, and recovery roadmap.

## Use when

Use for turnaround, cost reduction, margin improvement, cash improvement, operational reset, performance recovery, and urgent executive action plans.

## Do not use when

Do not use for normal operating plan updates, investor pitches, or long-range transformation unless performance recovery is the urgent decision.

## Storyline pattern

Cover, turnaround answer, performance gap, root causes, quick wins and structural levers, cash/control risks, roadmap, decision ask.

## Layout + component stack

Uses `metric-strip`, `cause-effect`, `portfolio-prioritization-matrix`, `sensitivity-grid`, `risk-matrix-register`, `phase-roadmap`, and `decision-log`.

## Theme / assets

Theme: `assets/themes/red.css`. No required media assets.

## Source inspiration

Source-neutral turnaround and performance improvement patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with performance gaps, cost or margin data, root causes, improvement levers, cash/control risks, scenarios, timeline, and decision asks.

## Flexible swaps

Swap portfolio matrix for ranked bars when levers are numeric, replace cause-effect with issue-tree logic when there are multiple branches, and replace sensitivity with risk register when uncertainty is mostly operational.

## When to diverge

Diverge when the user needs an operating plan, pricing strategy, or transformation program rather than urgent performance improvement.

## Do not copy page order/content

Do not copy the archetype page order, placeholder levers, footer text, or turnaround labels. Use it only to understand recovery-story composition.
