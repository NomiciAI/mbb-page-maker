AI application software investment thesis
Market structure

Application value is shifting from generic copilots toward vertical and workflow systems

Layer 1

Horizontal productivity

Broad assistants that sit beside knowledge work and compete on reach, admin control, and suite integration.

BuyerCIO, function leaders, productivity budget owners
WorkflowsMeeting notes, document drafting, research, inbox triage, personal task automation
MoatDistribution and suite integration; weak standalone switching cost
RiskHigh bundling risk as platforms absorb common use cases
StanceAvoid standalone wrappers unless they own a privileged channel
Layer 2

Functional workflow AI

Applications embedded in a functional operating rhythm, with measurable KPIs and repeatable deployment playbooks.

BuyerCRO, COO, CFO, GC, CTO, and functional operations leaders
WorkflowsPipeline creation, agent assist, finance close, contract review, code review
MoatUsage telemetry, integration depth, outcome-specific evaluation data
RiskCrowded GTM and partial platform adjacency in each function
StanceSource selectively; require proof of budget conversion and expansion
Layer 3

Vertical AI systems

Domain-specific operating systems for regulated, document-heavy, expert-labor, or service-intensive categories.

BuyerBusiness-unit owner with P&L pressure and compliance accountability
WorkflowsCompliance review, claims, clinical administration, specialty support operations
MoatDomain corpus, audit trails, workflow ownership, switching cost
RiskLonger sales cycles and heavier domain-specific implementation
StancePriority invest when onboarding becomes repeatable without losing depth
