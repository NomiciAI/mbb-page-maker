# CXO SteerCo Progress Update

## Summary

CXO, SteerCo, LOP, board-adjacent, and work-progress update full-deck archetype for separating status, evidence, risks, blockers, and decisions.

## Visual traits

Green operating update style, KPI strip, milestone status, risk/blocker register, decision log, and forward plan.

## Use when

Use for SteerCo updates, CXO progress reviews, leadership operating plan updates, board-adjacent status reads, and PMO progress packs.

## Do not use when

Do not use for deep strategy recommendation, workshop facilitation, or technical implementation design unless status and decisions are the main job.

## Storyline pattern

Cover, executive status, KPI snapshot, progress vs plan, risks and blockers, decision asks, forward plan, appendix/open items.

## Layout + component stack

Uses `metric-strip`, `milestone-track`, `risk-matrix-register`, `decision-log`, `phase-roadmap`, and `dense-list`.

## Theme / assets

Theme: `assets/themes/green.css`. No required media assets.

## Source inspiration

Source-neutral steering committee and executive update patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with KPI status, milestones, risks, blockers, decisions, owners, timing, and forward plan.

## Path

`templates/full-decks/cxo-steerco-progress-update/index.html`

## Related showcase patterns

`templates/showcase/steerco-progress-decision-page.html`, `templates/showcase/roadmap-risk-decision-page.html`, `templates/showcase/text-patterns.html`

## Flexible swaps

Swap milestone track for roadmap when future phasing matters, replace risk matrix with dense blocker list for tactical updates, and replace metric strip with table when exact status fields matter.

## When to diverge

Diverge when the audience needs a board narrative, a PMO task tracker, or a strategy recommendation rather than a progress-and-decisions update.

## Do not copy page order/content

Do not copy the archetype page order, placeholder metrics, footer text, or status labels. Use it only to understand update pacing and decision framing.
