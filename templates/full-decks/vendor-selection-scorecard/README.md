# Vendor Selection Scorecard

## Summary

Vendor or solution selection full-deck archetype for moving from requirements to shortlist, weighted scorecard, risk/TCO readout, and recommendation.

## Visual traits

Blue analytical selection style, criteria-first logic, weighted scorecards, implementation fit checks, risk register, and decision close.

## Use when

Use for vendor selection, platform evaluation, solution shortlist, RFP synthesis, or build/buy recommendation decks.

## Do not use when

Do not use for technical implementation sequencing after the vendor has already been selected.

## Storyline pattern

Cover, decision answer, requirements, shortlist, weighted scorecard, TCO/risk, implementation fit, recommendation, and next steps.

## Layout + component stack

Uses `metric-strip`, `weighted-scorecard`, `comparison-table`, `risk-matrix-register`, `phase-roadmap`, and `decision-log`.

## Theme / assets

Theme: `assets/themes/blue.css`. No required media assets.

## Source inspiration

Source-neutral vendor selection and RFP synthesis patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with requirements, vendor options, criteria weights, scores, commercial terms, implementation constraints, and decision owners.

## Path

`templates/full-decks/vendor-selection-scorecard/index.html`

## Related showcase patterns

`templates/showcase/vendor-scorecard-decision-page.html`, `templates/showcase/evidence-table-implication-page.html`, `templates/showcase/roadmap-risk-decision-page.html`

## Flexible swaps

Use `comparison-table` when weights are not defensible, `pros-cons-table` when the evidence is qualitative, and `risk-matrix-register` when implementation risk dominates score differences.

## When to diverge

Diverge when the user needs an implementation plan, negotiation pack, or detailed RFP appendix rather than a recommendation deck.

## Do not copy page order/content

Do not copy the archetype page order, placeholder scores, footer text, or vendor labels. Use it only to understand selection-story pacing and component combinations.
