# Advanced Analytics Readout

## Summary

Advanced analytics readout full-deck archetype for explaining a business question, method, data foundation, patterns, drivers, scenario sensitivity, and actions.

## Visual traits

Monochrome analytical style, table/chart density, driver ranking, segment comparison, sensitivity grid, and implication-led close.

## Use when

Use for data analysis readouts, model result explanations, cohort/segment analysis, driver analysis, scenario analysis, or analytics-to-action decks.

## Do not use when

Do not use for generic strategy decks or executive updates unless the core work is explaining analytical results.

## Storyline pattern

Cover, question and answer, method/data foundation, headline patterns, driver analysis, segment/cohort readout, sensitivity, implications, and action plan.

## Layout + component stack

Uses `metric-strip`, `comparison-table`, `ranked-bars`, `scatter-plot`, `sensitivity-grid`, `decision-log`, and `callout`.

## Theme / assets

Theme: `assets/themes/mono.css`. No required media assets.

## Source inspiration

Source-neutral analytics readout and insight-to-action patterns. No external deck, brand system, or consulting-firm identity is copied.

## Evidence shape

Best with numeric outputs, drivers, segments, cohorts, scenarios, caveats, confidence levels, and action implications.

## Path

`templates/full-decks/advanced-analytics-readout/index.html`

## Related showcase patterns

`templates/showcase/analytics-driver-insight-page.html`, `templates/showcase/evidence-table-implication-page.html`, `templates/showcase/executive-answer-data-page.html`

## Flexible swaps

Swap ranked bars for tables when exact values matter, use scatter plots for two-axis outputs, and replace sensitivity grids with callouts when scenarios are not quantified.

## When to diverge

Diverge when the user needs a dashboard, appendix-heavy model documentation, or strategy recommendation rather than analytical explanation.

## Do not copy page order/content

Do not copy the archetype page order, placeholder values, footer text, or analysis labels. Use it only to understand analytics-readout pacing and component combinations.
